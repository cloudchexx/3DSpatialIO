# 三维空间数据高效读写 — 开发文档

---

## 一、项目概述

设计一个**无冗余、高均衡、高性能**的三维数据存储引擎。核心目标：在不增加冗余副本的前提下，使 X、Y、Z 三个维度的切片读取速度均衡且高效。

---

## 二、核心约束

| 约束项 | 要求 |
|--------|------|
| 精度 | 单点相对误差 < 1‰（一票否决） |
| 存储膨胀 | ≤ 原始数据 1.5 倍（满分 20） |
| 内存上限 | ≤ 2GB（超过可忽略该限制，但仍需可配置） |
| 并发模型 | 单进程多线程，无多节点 |
| 语言 | C/C++ 优先，可 CUDA 加速 |
| 开放库 | 仅限开源免费库，禁商用闭源 |

### 7.3 算法流程

候选池 `{16, 32, 64, 128, 256, 512}` 三层穷举 (216 种组合)，依次通过：

1. **体积约束过滤** — 单块 512KB ~ 4MB（以 float 为例），避免索引膨胀和 I/O 粒度失衡
2. **存储红线过滤（一票否决）** — Padding 膨胀率 < 1.45（留 5% 裕量），`int64_t` 防溢出
3. **内存连续性感知评分** — `cost = max(Tx,Ty,Tz) / min(Tx,Ty,Tz)`，选 cost 最小者

得到满足 $c_x > c_y > c_z$ 逆向补偿策略的最优 Chunk 形状。

## 七-B、大文件流式改造 — 已完成 ✅

> 详细方案见 [`大文件流式改造实施方案.md`](./大文件流式改造实施方案.md)

### 7B.1 背景与目标

原有架构存在 5 个 32 位/全量内存瓶颈：
1. `fseek`/`ftell` 返回 `long`（Win 32 位上限 2GB）
2. `out_data.resize(num_floats)` 一次性全量分配
3. `ftell(f)` 跟踪输出偏移（>2GB 溢出）
4. `fseek(static_cast<long>(offset))` 偏移截断
5. 全量 `vector<float> data` 绑定全流程

**改造目标**：支持 TB 级数据包，内存占用 ≤ 2GB。

### 7B.2 核心改动

| # | 模块 | 改动 |
|---|------|------|
| 1 | `include/file_io.h` | 新增 `RawFileReader` 类（流式读取器）+ 跨平台 `FSEEK64`/`FTELL64` 宏 |
| 2 | `src/file_io.cpp` | 实现 `open()`（64 位文件大小校验 + `skip_bytes` 支持）、`read_region()`（按区域流式读取）、`close()` |
| 3 | `include/chunk_io.h` | 新增 `write_c3dr_file_stream()` 声明；`C3DRIndexEntry::offset` 改为 `int64_t`；引入 `file_io.h` |
| 4 | `src/chunk_io.cpp` | 实现子组流式写入算法；修复 `ftell` → 公式计算；修复 `fseek` → `FSEEK64`；修复写入排序 bug；优化 I/O 为大块连续读 |
| 5 | `src/main.cpp` | 替换 `vector<float> data` → `RawFileReader`；流式流程：open → optimize → stream_write |
| 6 | `src/profiler.cpp` | `benchmark_slices()` 适配 `RawFileReader&`，移除全量数据依赖 |
| 7 | `src/args.cpp` | 新增 `--output`、`--elem-size`、`--skip-bytes`、`--profile`、`--benchmark` 参数 |

### 7B.3 子组流式写入算法

```
FOR iy = 0 .. nc_y-1:
    FOR ix = 0 .. nc_x-1:
        // 步骤 A：每 x 层一次大块 fread（~640KB），分散到 nc_z 个 chunk 缓冲区
        FOR lx = 0 .. x_len-1:
            fread(y_row_buf)  // 一次性读取固定 x 下 y 区间全部 z 数据
            FOR iz = 0 .. nc_z-1:
                memcpy chunk_bufs[iz] ← y_row_buf 对应片段

        // 步骤 B：FSEEK64 定位后按 chunk_idx 写入各自偏移
        FOR iz = 0 .. nc_z-1:
            fseek(data_offset + chunk_idx * chunk_bytes)
            fwrite(chunk_bufs[iz])

回写 Header + IndexTable
```

**内存占用**：仅 `nc_z × cx × cy × cz × 4` + `IndexTable`，典型 < 100 MB。

### 7B.4 过程中修复的关键 Bug

| Bug | 描述 | 修复 |
|-----|------|------|
| 写入顺序错位 | 子组遍历 `(iy, ix)` ≠ chunk_idx 线性顺序，顺序 `fwrite` 导致 chunk 数据写入错误偏移 | 步骤 B 每次写入前显式 `FSEEK64` 到正确位置 |
| I/O 性能瓶颈 | 原始逐 `(x,y)` 对随机寻址 → 18GB 文件需 ~8000 万次 seek | 改为每 x 层一次大块连续 `fread`（~640KB），降至 ~1.5 万次 seek（~5000× 提升） |

### 7B.5 18GB 实测结果


## 八、任务 3：智能调度 — 已完成 ✅

> 详细方案见 [`智能调度Cache层实施方案.md`](./智能调度Cache层实施方案.md)

### 8.1 目标

构建 Chunk 级 LRU 缓存层，支持读写切面：
- 读命中零拷贝（`shared_ptr<const>` 引用计数递增）
- 写就地修改（`use_count==1`）或 Copy-on-Write（有读者持有）
- 脏块延迟落盘（Write-Back）
- 连续模式预取

### 8.2 架构总览

```
C3DRCacheReader（继承 C3DRReader）
  │
  ├─ [Read] read_chunk_by_index(idx) override
  │   ├─ 命中 → shared_ptr<const> 零拷贝 + LRU 前移
  │   └─ 未命中 → 父类磁盘 I/O → 插入缓存
  │
  ├─ [Write] write_x/y/z_slice → 逐 chunk 修改
  │   ├─ use_count==1 → 就地修改（零拷贝写）
  │   ├─ use_count>1  → COW：拷贝→修改→替换 shared_ptr
  │   └─ 不在缓存 → 加载→插入缓存→修改
  │   全部标记 is_dirty=true
  │
  ├─ [Flush] 脏块 → FSEEK64+fwrite → is_dirty=false
  ├─ [Evict] LRU 尾淘汰 → 脏块先 flush → 再移除
  └─ maybe_prefetch()（仅读路径触发，连续 3+ 同轴切片时预取）
```

### 8.3 核心设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 写策略 | Write-Back + COW 回退 | use_count==1 就地修改（零拷贝写），有读者时 COW（安全）|
| 缓存粒度 | Chunk 级 | 一个 chunk 可被 X/Y/Z 三轴切面复用，无冗余 |
| 淘汰策略 | LRU + 预取 | 连续切片天然局部性好；LRU 实现简单高效 |
| 多线程模型 | shared_mutex + io_mutex | 命中走 shared_lock，未命中/写走 unique_lock |
| 数据管理 | shared_ptr\<const vector\<float\>\> | 写时 COW 替换整个 shared_ptr，旧 ptr 读者安全 |
| 文件模式 | `r+b` | C3DRCacheReader::open() 重新以读写模式打开 |

### 8.6 线程安全模型

```
m_cache_mutex (shared_mutex)
┌──────────────────────────────────┐
│ 读切片: shared_lock → shared_ptr │ → 释放锁后安全读
│ 写切片: unique_lock + COW        │ → use_count==1 就地改，>1 拷贝替换
│ Flush:  unique_lock + io_mutex   │ → 脏块落盘
└──────────────────────────────────┘
```

### 11.4 bench_e2e.cpp 核心逻辑

```cpp
int main(int argc, char* argv[]) {
    // 解析参数: c3dr_path, output_path, cache_size_mb
    
    C3DRCacheReader reader(cache_bytes);
    reader.open(c3dr_path);
    
    FILE* fout = fopen(output_path.c_str(), "wb");
    std::mt19937 rng(42);  // 固定种子可复现
    
    // ── 随机切片 100 次/轴 ──
    for (int i = 0; i < 100; ++i) {
        uint32_t x = rng() % dim_x;
        auto slice = reader.read_x_slice(x);
        fwrite(slice.data(), 4, slice.size(), fout);
    }
    // Y/Z 轴同理...
    
    // ── 连续切片 10 次/轴 ──
    uint32_t start_x = rng() % (dim_x - 10);
    for (int i = 0; i < 10; ++i) {
        auto slice = reader.read_x_slice(start_x + i);
        fwrite(slice.data(), 4, slice.size(), fout);
    }
    // Y/Z 轴同理...
    
    // 输出统计报告
    printf("随机切片: X=%.2fms Y=%.2fms Z=%.2fms\n", ...);
    printf("连续切片: X=%.2fms Y=%.2fms Z=%.2fms\n", ...);
    printf("缓存命中率: %.1f%%\n", hit_rate);
}
```
