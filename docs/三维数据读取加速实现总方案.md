# 三维空间数据高效读写 — 开发文档

---

## 一、项目概述

设计一个**无冗余、高均衡、高性能**的三维数据存储引擎。核心目标：在不增加冗余副本的前提下，使 X、Y、Z 三个维度的切片读取速度均衡且高效。

---

## 二、核心约束

| 约束项 | 要求 |
|--------|------|
| 精度 | 单点相对误差 < 1‰（一票否决） |
| 存储膨胀 | ≤ 原始数据 1.5 倍（满分 20） |
| 内存上限 | ≤ 2GB（超过可忽略该限制，但仍需可配置） |
| 并发模型 | 单进程多线程，无多节点 |
| 语言 | C/C++ 优先，可 CUDA 加速 |
| 开放库 | 仅限开源免费库，禁商用闭源 |

---

## 三、评分拆解

| 评分项 | 分值 | 关键点 |
|--------|------|--------|
| 功能指标 | 否决项 | 读写结果准确，误差 < 1‰ |
| 性能指标 | 60 | X/Y/Z 三维随机切片(100次) + 连续切片(10次)，读完后写出标准格式，综合用时越短越好 |
| 存储指标 | 20 | 存储 ≤ 原始 1.5 倍满分，每超 10% 扣 1 分 |

## 六、任务 1：接管数据 — 已完成

### 6.1 功能说明

接收用户原始三维浮点数据（row-major 二进制 float32 格式），验证文件完整性，计算基本统计信息。

### 6.2 已实现模块

#### `include/common.h` — 全局常量与数据结构

定义文件格式版本号 `C3DR_VERSION = 1`、`ChunkShape` 结构体 `(cx, cy, cz)`、候选集配置 `CANDIDATES[]`、体积区间 `MIN/MAX_CHUNK_BYTES`、存储红线 `MAX_STORAGE_RATIO = 1.45`、代价权重 `W_IO / W_MEM` 等。

#### `include/args.h` + `src/args.cpp` — CLI 参数解析器

| 参数 | 说明 |
|------|------|
| `-i`, `--input <path>` | 原始 float32 二进制文件路径 |
| `--dim-x <N>` | 数据 X 维度尺寸 |
| `--dim-y <N>` | 数据 Y 维度尺寸 |
| `--dim-z <N>` | 数据 Z 维度尺寸 |
| `-h`, `--help` | 显示帮助信息 |

```cpp
struct Args {
    std::string input_path;
    uint32_t    dim_x = 0;
    uint32_t    dim_y = 0;
    uint32_t    dim_z = 0;
    bool        show_help = false;
};
```

#### `include/file_io.h` + `src/file_io.cpp` — 原始文件读取器

```cpp
struct RawDataInfo {
    uint32_t dim_x, dim_y, dim_z;
    size_t   total_elements;
    float    value_min, value_max;
};

RawDataInfo read_raw_float_file(
    const std::string& path,
    uint32_t dim_x, uint32_t dim_y, uint32_t dim_z,
    std::vector<float>& out_data);
```

- 以二进制方式打开文件，验证文件大小 = `dim_x × dim_y × dim_z × sizeof(float)`
- 一次性读入 `std::vector<float>`
- 单次遍历计算 value_min/value_max，若数据为空则均为 0.0
- 任何异常（无法打开、尺寸不匹配、读取不完整）抛出 `std::runtime_error`

#### `src/main.cpp` — 程序入口

流程：
1. 解析 CLI 参数
2. 调 `read_raw_float_file()` 读入数据
3. 输出文件路径、维度、元素数、数据量(MB)、值域
4. 计算 rolling hash 校验和（multiply-by-31 over float bits）
5. 捕获异常并输出错误信息

### 6.3 编译与运行

```bash
# 配置 (在 build/ 目录中)
cmake -B build -S .
# 编译
cmake --build build --config Release
# 运行
./build/bin/Release/cc_3Dreader --input test_input.bin --dim-x 100 --dim-y 200 --dim-z 50
```

### 7.1 实现概览

自适应切块算法已完整实现，涵盖四个阶段：

| 阶段 | 内容 | 关键文件 |
|------|------|----------|

| Phase 4 | 集成与 Profiling | `include/chunk_io.h`、`src/chunk_io.cpp`、`include/profiler.h`、`src/profiler.cpp` |

### 7.2 已实现模块

| 模块 | 文件 | 说明 |
|------|------|------|
| `ChunkShape` | `include/common.h:9` | 最优切块尺寸 `(cx, cy, cz)`，三个 `int` 成员 |
| `C3DRHeader` | `include/chunk_optimizer.h:10` | `.c3dr` 文件头（magic、version、dim、chunk_shape、index_offset、data_offset），64 字节对齐 |
| `ChunkOptimizer::find_optimal_chunk_shape()` | `include/chunk_optimizer.h:19`、`src/chunk_optimizer.cpp` | 自适应切块算法核心：候选池 {16,32,64,128,256,512} 三层穷举 (216 组合)，经体积约束 → 存储红线 → 内存连续性评分，选出最优形状 |
| `write_c3dr_file()` | `include/chunk_io.h`、`src/chunk_io.cpp` | 按最优 ChunkShape 动态分配缓冲区，逐块写入 `.c3dr`，边界块自动补零 |
| `C3DRReader` | `include/chunk_io.h`、`src/chunk_io.cpp` | 从 Header 读取 chunk_shape，按需提取 X/Y/Z 任意维度切面 |
| Profiler | `include/profiler.h`、`src/profiler.cpp` | `calibrate_w_mem()` 分配 1GB 内存标定真实 ω_mem；`benchmark_slices()` 落盘后分别沿 X/Y/Z 读中间层切面，记录三轴耗时及均衡比 |

### 7.3 算法流程

候选池 `{16, 32, 64, 128, 256, 512}` 三层穷举 (216 种组合)，依次通过：

1. **体积约束过滤** — 单块 512KB ~ 4MB（以 float 为例），避免索引膨胀和 I/O 粒度失衡
2. **存储红线过滤（一票否决）** — Padding 膨胀率 < 1.45（留 5% 裕量），`int64_t` 防溢出
3. **内存连续性感知评分** — `cost = max(Tx,Ty,Tz) / min(Tx,Ty,Tz)`，选 cost 最小者

得到满足 $c_x > c_y > c_z$ 逆向补偿策略的最优 Chunk 形状。
