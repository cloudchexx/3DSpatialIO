# 三维数据切块存储优化方案

> **摘要**：本文档提出面向大规模三维科学数据（地震波、医学影像等）的"硬件感知自适应切块算法"。方案以有界网格搜索为框架，以内存在连续性感知代价函数为核心评价标准，在满足"存储膨胀 ≤ 1.5 倍"硬约束的前提下，实现 X / Y / Z 三轴切面读取的 I/O 性能均衡。

---

## 一、背景与问题

实际三维数据（物探地震波、医学影像等）绝大多数为**各向异性长方体**，维度比例可能极为悬殊。例如地震数据 X/Y 测线方向可达数千至数万采样点，而 Z 方向仅数百层。

若采用写死的固定正方体切块（如 $64 \times 64 \times 64$），会引发两个致命问题：

| 问题 | 说明 |
|---|---|
| **存储膨胀失控** | 边界 Padding 浪费严重。极端场景（如 $10000 \times 10000 \times 10$ 数据，Z 轴仅 10 层却分配 64 层存储），Z 轴浪费 84%，存储膨胀数倍，直接突破 1.5 倍红线 |
| **索引膨胀** | 若切块过小（几 KB），产生数亿个索引节点，索引自身占用数百 MB ~ 上 GB，突破 2GB 内存限制 |

**核心需求**：根据原始数据 shape 动态计算最优切块尺寸 $(c_x, c_y, c_z)$，同时满足三个目标：

1. **存储红线**：Padding 膨胀率 < 1.5 倍（留 5% 裕量，取 1.45 倍）
2. **硬件对齐**：单块体积适配内存页大小（4KB 倍数），如 512KB ~ 4MB
3. **三维读写均衡**：X/Y/Z 三轴综合耗时（I/O + 内存重组）极差最小

## 二、算法总体架构

算法由两个阶段构成：

```
原始数据 shape (X, Y, Z)
        │
        ▼
┌─────────────────────────┐
│  第一阶段：有界网格搜索    │  ← 遍历候选池，硬约束过滤
│  · 候选池生成             │
│  · 体积约束过滤            │
│  · 存储红线过滤            │
└──────────┬──────────────┘
           │ 合法候选组合
           ▼
┌─────────────────────────┐
│  第二阶段：内存连续性感知  │  ← 代价函数评分，选出最优
│  · 宏观 I/O 代价计算       │
│  · 微观内存跳跃代价计算    │
│  · 均衡度评分 (Cost)       │
└──────────┬──────────────┘
           │
           ▼
    输出最优 (cx, cy, cz)
```

---

## 三、第一阶段：有界网格搜索

### 3.1 候选池生成

限制 $c_x, c_y, c_z$ 在 $\{16, 32, 64, 128, 256, 512\}$ 中取值，确保尺寸为 $2^n$，利于底层内存对齐（Cache Line 64B、Page Size 4KB、CUDA Warp Size 32）。

三层循环穷举，共 $6^3 = 216$ 种组合，搜索耗时 < 1ms。

### 3.2 体积约束过滤

剔除单块体积累不合理的组合：

$$
512\text{KB} \le c_x \times c_y \times c_z \times 4 \le 4\text{MB} \quad (\text{以 float 为例})
$$


- **下限**：避免块数过多导致索引表膨胀
- **上限**：避免单次 I/O 粒度过大，浪费带宽和缓存

### 3.3 存储红线过滤（一票否决）

$$
\text{Ratio} = \frac{\lceil X/c_x \rceil c_x \cdot \lceil Y/c_y \rceil c_y \cdot \lceil Z/c_z \rceil c_z}{X \cdot Y \cdot Z}
$$

若 $\text{Ratio} > 1.45$，直接淘汰。

## 四、第二阶段：内存连续性感知代价函数

### 4.1 核心原理：逆向补偿

C/C++ 三维数组按行优先（Row-Major）展开，Z 轴为最内层、物理连续；X 轴为最外层、高度离散。

| 切面方向 | 内存跳跃次数 | 连续拷贝长度 | 速度 |
|---|---|---|---|
| X 切面（固定 x） | $c_y$ | $c_z$ | 最快 |
| Y 切面（固定 y） | $c_x$ | $c_z$ | 中等 |
| Z 切面（固定 z） | $c_x \times c_y$ | 1 | 最慢 |

**逆向补偿策略**：

- **Z 轴（最内层，微观最快）**：刻意减小 $c_z$，用更少的宏观 Chunk 数量弥补微观 Cache Miss
- **X 轴（最外层，微观最慢）**：刻意增大 $c_x$，虽然宏观 Chunk 数增多，但因微观提取极快，整体可控

因此最优均衡形状通常满足 $c_x > c_y > c_z$（如 $128 \times 64 \times 32$），而非正方体。

### 4.2 代价函数公式

**宏观 I/O 代价（读取块数）**：
$$
\begin{aligned} N_x &= \left\lceil\frac{Y}{c_y}\right\rceil \cdot \left\lceil\frac{Z}{c_z}\right\rceil \\ N_y &= \left\lceil\frac{X}{c_x}\right\rceil \cdot \left\lceil\frac{Z}{c_z}\right\rceil \\ N_z &= \left\lceil\frac{X}{c_x}\right\rceil \cdot \left\lceil\frac{Y}{c_y}\right\rceil \end{aligned}
$$
**融合微观内存代价的耗时估算**：
$$
\begin{aligned} T_x &= N_x \cdot \big(\omega_{io} + c_y \cdot \omega_{mem}\big) \\ T_y &= N_y \cdot \big(\omega_{io} + c_x \cdot \omega_{mem}\big) \\ T_z &= N_z \cdot \big(\omega_{io} + c_x \cdot c_y \cdot \omega_{mem}\big) \end{aligned}
$$
其中：
- $\omega_{io}$：单块磁盘 I/O 耗时权重（经验值 1.0）
- $\omega_{mem}$：单次非连续内存跳跃开销（经验值 0.005，需根据实际机器 profiling 微调）

**均衡度评分**：
$$
\text{Cost} = \frac{\max(T_x, T_y, T_z)}{\min(T_x, T_y, T_z)}
$$
Cost 越接近 1，三个方向的读写性能越均衡。

### 4.3 伪代码

```cpp
struct ChunkShape { int cx, cy, cz; };

ChunkShape find_optimal_chunk_shape(int data_X, int data_Y, int data_Z) {
    std::vector<int> candidates = {16, 32, 64, 128, 256, 512};
    ChunkShape best_shape = {64, 64, 64};
    double min_cost = std::numeric_limits<double>::max();

    const double W_io  = 1.0;
    const double W_mem = 0.005;

    for (int cx : candidates) {
        for (int cy : candidates) {
            for (int cz : candidates) {
                // 硬约束：体积
                long long vol = (long long)cx * cy * cz * 4;
                if (vol < 512 * 1024 || vol > 4 * 1024 * 1024) continue;

                // 硬约束：存储红线
                long long pad_x = ((data_X + cx - 1) / cx) * cx;
                long long pad_y = ((data_Y + cy - 1) / cy) * cy;
                long long pad_z = ((data_Z + cz - 1) / cz) * cz;
                double ratio = (double)(pad_x * pad_y * pad_z)
                                / ((long long)data_X * data_Y * data_Z);
                if (ratio > 1.45) continue;

                // 宏观 I/O 块数
                long long Nx = ((data_Y + cy - 1) / cy) * ((data_Z + cz - 1) / cz);
                long long Ny = ((data_X + cx - 1) / cx) * ((data_Z + cz - 1) / cz);
                long long Nz = ((data_X + cx - 1) / cx) * ((data_Y + cy - 1) / cy);

                // 内存连续性感知耗时
                double Tx = Nx * (W_io + cy * W_mem);
                double Ty = Ny * (W_io + cx * W_mem);
                double Tz = Nz * (W_io + (cx * cy) * W_mem);

                double max_T = std::max({Tx, Ty, Tz});
                double min_T = std::min({Tx, Ty, Tz});
                double cost = max_T / min_T;

                if (cost < min_cost) {
                    min_cost = cost;
                    best_shape = {cx, cy, cz};
                }
            }
        }
    }
    return best_shape;
}
```

## 五、工程实施建议

### 5.1 集成流程

```
用户导入数据 (shape: X, Y, Z)
    │
    ▼
ChunkOptimizer::find_optimal_chunk_shape(X, Y, Z)   ← 1ms 内完成
    │
    ▼
写入自定义格式 Header（存储 cx, cy, cz）
    │
    ▼
切分 / 落盘 / CUDA 还原 → 全部从 Header 读取这三个参数
```

### 5.2 参数调优指南

| 参数 | 默认值 | 调优方法 |
|---|---|---|
| 候选集 | $\{16, 32, 64, 128, 256, 512\}$ | 可根据目标硬件调整，保持 $2^n$ |
| 体积区间 | 512KB ~ 4MB | 以单次 SSD 4KB 页读取对齐为目标 |
| $\omega_{io}$ | 1.0 | 固定为 1.0 作为基准 |
| $\omega_{mem}$ | 0.005 | 通过本机 profiling 实测：分别测连续大块读取 vs 随机小读取的耗时比 |
| 存储裕量 | 1.45 | 若实测 Padding 可控可放宽至 1.48 |

### 5.3 优势总结

| 维度 | 效果 |
|---|---|
| **存储** | 硬约束保证膨胀率 < 1.45 倍，满足赛题 1.5 倍红线 |
| **性能** | 逆向补偿策略使三轴读取耗时极差最小化，均衡读写 |
| **内存** | 候选池限制（体积下限 512KB）控制索引节点数量，满足 2GB 限制 |
| **硬件友好** | 尺寸均为 $2^n$，天然对齐 Cache Line / Page Size / Warp Size |
| **计算开销** | 搜索空间 216 种组合，< 1ms，可在 import 阶段实时完成 |
