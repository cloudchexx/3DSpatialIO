# 三维空间数据高效读写赛题文档

---

## 一、项目背景

在科学计算、数值模拟、三维可视化、医学影像、空间测绘与海量三维数据交互分析等领域，规则三维数据体已成为核心数据形态。这类数据在空间上由 X、Y、Z 三个正交维度构成连续统一的整体，广泛应用于实时剖面浏览、多视角切片提取、局部精细查询等高频操作场景。

当前主流三维数据在磁盘上普遍采用单一维度顺序存储方式，即沿某一固定维度连续排布数据单元，其余维度则以跨块、非连续形式存在。这种组织方式在沿存储维度访问时可获得较高效率，但在沿另外两个维度进行剖面读取时，会产生大量磁盘随机 IO、频繁磁头跳转与离散寻址操作。随着数据规模持续扩大，单数据集已达到 TB 级，随机访问带来的延迟会被显著放大，导致数据加载卡顿、交互响应迟缓，无法满足高效分析与实时可视化的性能要求。

为缓解多维度访问效率低下的问题，传统方案通常采用多副本冗余存储，即沿 X、Y、Z 三个维度分别生成一份顺序存储文件，确保每个维度都能以连续方式读取。但该方法会使存储空间占用呈倍数增长，带来巨大的存储成本、数据同步与管理负担，在大规模数据场景下难以实用。

目前工业界与学术界的高效三维数据系统普遍转向优化存储结构 \+ 精细化索引方案来提升数据访问效率。这种方案无需多副本，即可显著提升三个维度的访问均衡性，在存储效率与 IO 性能之间取得最优平衡。

为此，本项目聚焦无冗余、高均衡、高性能的三维数据组织方式，要求设计一种面向三维数据的高效存储结构与配套索引机制，通过物理布局优化、切片预对齐、快速定位索引等手段，实现在不增加冗余存储的前提下，X、Y、Z 三个方向均具备接近连续读取的高效读写能力，为大规模三维数据的实时交互与高效处理提供稳定支撑。

## 三、项目需求

1. 研究面向多维度高效访问的三维数据存储结构及配套高速读写算法库，实现原始浮点数组与优化存储格式数据的双向转换；

2. 算法库支持三维数据沿 X、Y、Z 三个维度的任意切片访问，三个维度访问效率均衡；

3. 算法库支持三维数据体按主维度（如 X）的单列高效读取；

4. 算法库在单机环境下运行，采用单进程多线程模式，不采用多节点并行；

5. 算法库可使用 CUDA 等通用 GPU 加速技术提升处理速度；

6. 算法内存使用量可配置、可管控，支持通过参数设置最大可用内存，避免过度占用系统内存（内存使用≤2GB 时可忽略该限制）；

7. 程序优先使用 C/C\+\+ 语言开发，允许使用开源免费的可视化算法库，禁止使用商用闭源库。

## 四、评分标准

|---|---|---|---|---|
|1|功能指标|否决项|数据读写结果准确，完整还原原始三维数据，单点相对误差小于千分之一；不满足则不得分。|\-|
|2|性能指标|60|程序运行中读取时间越短，得分越高，采用 X、Y、Z 三个维度读取用时综合计算；各维度切片读取用时占比均为 1/3，综合得分为 `\(基准时间 / 实际综合时间\) × 60`；测试各维度的随机切片读取（100 次随机坐标切片）、连续切片读取（10 次连续大切片），取平均用时，读取结束的标志为完成写出为标准格式。|60|
|3|存储指标|20|存储结构紧凑，无冗余副本，存储空间使用小于原始数据的 1\.5 倍得 20 分，占用空间每超 10%，扣 1 分。|20|
